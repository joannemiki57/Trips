핵심 규칙 요약

① 사진은 기기에 그대로 — 원본 바이트는 저장 안 함. assetLocalId만 저장 (mvp #8). 사진 100GB여도 우리 DB는 수 MB.
② CloudKit 동기화 = 하이브리드 (5차 라운드) — 사용자 수동 수정(Scene 합치기/나누기·메모·label·♥·Trip 정의)은 동기. 자동 묶음(B3 알고리즘 결과)은 기기별 재계산. 충돌 표면 최소.
③ Day는 필수, Scene은 선택 — Photo 생성 시 init(day:)로 강제. 단일 사진 토글 OFF가 기본 → Photo가 Day에 직접 소속.
④ ♥ 누르면 자동 승격 — Scene.representativePhoto가 즉시 교체. Visual Spine 마디 두께도 ♥ 개수 따라.
⑤ 사진 사라지면 (C3) — Photo 레코드는 남기되 isMissing=true + lastVerifiedAt 캐시. memo/label/♥ 보존. 매번 PhotoKit 조회 비용 회피. 다른 사진으로 이관은 v1.1.
⑥ 동기화 충돌 — v1은 last-write-wins (C6).
⑦ Label 전파 — Scene에 붙은 라벨이 자동으로 Photo에 저장되진 않음. 쿼리 시 union (photo.allLabels = labels ∪ scene?.labels).
⑧ 점수 캐시 — scoreVersion이 현재 알고리즘 버전과 다르면 무시·재계산. 산식 바뀌어도 점진 마이그레이션.
⑨ 타임존 — Day 경계는 촬영 시점 EXIF tz 기준. EXIF tz 없으면 GPS→tz, 그것도 없으면 사용자 현재 tz로 폴백.
⑩ 03:00 룰 재계산 — Photo 추가/삭제/이동 시 그 사진의 capturedAt이 02:00~04:00 구간일 때만 인접 Day 두 개 재평가. 전체 재계산 X.
